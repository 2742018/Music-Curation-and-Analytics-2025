Formatting Considerations

At this stage, the metadata is displayed as unformatted XML text above the Verovio-rendered score. This presentation is difficult for non-specialist users to read and does not reflect typical archival or catalogue standards. To make the metadata more usable and visually coherent, I would implement several formatting improvements:

  • Apply CSS styling to visually distinguish key metadata categories (e.g., title, composer, genre, licence). This would allow each field to be presented with appropriate hierarchy, spacing, and emphasis.
  • Convert the metadata into structured blocks or card-style catalogue entries, mirroring the format used in digital library interfaces. This would make the information easier to scan and interpret at a glance.
  • Use collapsible sections for longer or more technical fields, such as editorial notes or licensing statements. This maintains readability while still preserving all the details without taking up too much space.
  • Link authority-controlled names (e.g., Koji Kondo, Nintendo) to external databases such as VIAF, MusicBrainz, or the Zelda Wiki. This enhances discoverability and situates the metadata within wider information networks.

For this project, a CSS-based solution is the most practical. It is lightweight, reliable, and appropriate for a static archival website. CSS also maintains long-term browser compatibility, whereas JavaScript-based rendering would require ongoing maintenance and more complex scripting to dynamically parse the MEI header.
