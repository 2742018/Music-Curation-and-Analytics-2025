Metadata Revision & Web Rendering with Verovio

This week's task focused on revising and enriching the metadata of my MEI file and displaying it on a webpage using Verovio. I updated my Week 5 MEI file with improved metadata, including genre classification and explicit licensing, and embedded it in my website using the Verovio engine.

1. Revising My MEI Metadata

1.1 Creating the Updated MEI File

Using the original Week 5 MEI file, I saved a new version titled week7meiver2_zelda-great-fairy-fountain-piano-etude-music-by-koji-kondo-arrangement-by-erik-c-piano-man-page1.mei. This new version includes two main improvements: genre classification and explicit licensing information.

✔ Genre Classification Added:

<notesStmt>
   <note type="genre">
      Video game music; Piano arrangement; 1980s–present Nintendo franchise
   </note>
</notesStmt>

✔ Licensing Information (Creative Commons BY-NC 4.0):

<availability>
   <licence target="https://creativecommons.org/licenses/by-nc/4.0/">
      CC BY-NC 4.0 – This encoding may be shared and adapted for non-commercial educational use.
   </licence>
</availability>

1.2 Why These Metadata Choices?

Genre: I classified this piece as video game music because it is derived from Koji Kondo’s score for The Legend of Zelda. This franchise has played a central role in shaping Nintendo's musical identity since the 1980s. The secondary label piano arrangement is also important because the MEI file does not encode the original in-game chiptune audio, but a contemporary fan-created etude that stylistically expands upon Kondo’s motif. Specifying both the franchise context and the arranged medium allows users to understand better the piece's transmission history and how it moved from 8-bit sound hardware to a modern interpretive piano setting. This also shows the essential for digital musicology work that deals with versioning, arrangement lineage, and source transparency.

Licensing: I selected the Creative Commons CC BY-NC 4.0 licence because it balances accessibility with responsible reuse. The MEI file constitutes a derivative encoding that combines elements from a publicly shared fan arrangement with my own editorial corrections and manual corrections. As such, it should remain freely available for learning, teaching, and scholarly analysis, but should not be repurposed for commercial use. The BY-NC licence achieves this by allowing users to copy, distribute, and adapt the material provided that attribution is given and no commercial use is made. This aligns with typical practice in digital humanities archiving, where transparency, openness, and ethical sharing are prioritised while protecting creators from any unapproved monetisation of their contributions.

3. Formatting Considerations

At this stage, the metadata is displayed as unformatted XML text above the Verovio-rendered score. This presentation is difficult for non-specialist users to read and does not reflect typical archival or catalogue standards. To make the metadata more usable and visually coherent, I would implement several formatting improvements:

  • Apply CSS styling to visually distinguish key metadata categories (e.g., title, composer, genre, licence). This would allow each field to be presented with appropriate hierarchy, spacing, and emphasis.
  • Convert the metadata into structured blocks or card-style catalogue entries, mirroring the format used in digital library interfaces. This would make the information easier to scan and interpret at a glance.
  • Use collapsible sections for longer or more technical fields, such as editorial notes or licensing statements. This maintains readability while still preserving all the details without taking up too much space.
  • Link authority-controlled names (e.g., Koji Kondo, Nintendo) to external databases such as VIAF, MusicBrainz, or the Zelda Wiki. This enhances discoverability and situates the metadata within wider information networks.

For this project, a CSS-based solution is the most practical. It is lightweight, reliable, and appropriate for a static archival website. CSS also maintains long-term browser compatibility, whereas JavaScript-based rendering would require ongoing maintenance and more complex scripting to dynamically parse the MEI header.

4. Reflection

Updating my MEI metadata and rendering it on a webpage has strengthened my understanding of curatorial responsibility in digital music archiving. Metadata helps shape how future users interpret and ethically reuse the material. The Verovio interface demonstrates how encoded music can be made accessible, interactive, and meaningful across different levels of technical expertise.
